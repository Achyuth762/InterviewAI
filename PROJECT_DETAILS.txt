AI Mock Interview System - Technical Inventory

Generated: March 29, 2026

1) Architecture Overview
- Stack: MERN (MongoDB, Express, React, Node.js)
- Frontend: React 18 + Vite + Tailwind CSS + Framer Motion + Recharts
- Backend: Node.js + Express + MongoDB + Google Generative AI (Gemini)
- Auth: JWT + Google OAuth 2.0
- Core flow: Authentication -> Interview Setup -> Media Permissions -> Interview Session -> AI Evaluation -> Results/Analytics

2) Project Structure
- client/: React application (UI, routing, contexts, hooks, services)
- server/: Express API (routes, controllers, models, middleware, validators, AI service)
- package.json: root scripts for install/dev/build/seed/lint
- README.md: product and setup overview

5) Backend (server)
5.1) Server Startup and Middleware
- server/server.js
  - Loads env
  - Connects DB
  - Security middleware: helmet, mongo sanitize, cors, rate limiting
  - Parsers: express.json/urlencoded
  - Logging: morgan in dev
  - Mounts routes at /api/*
  - Health endpoint: GET /api/health
  - Not-found + centralized error handling middleware

5.2) Database
- server/config/db.js
  - Mongoose connection via MONGODB_URI
  - Logs success host and exits process on failure

5.3) Models
- server/models/User.js
  - Fields: name, email, password, googleId, authProvider, avatar, role, profile, preferences, stats
  - Password hashed with bcrypt pre-save
  - comparePassword method
  - toJSON strips password
  - Indexes on email and googleId
- server/models/Interview.js
  - Tracks interview lifecycle: in-progress/completed/abandoned
  - Stores answers array with score/feedback/strengths/improvements/timeSpent/skipped
  - Stores totals: totalScore/maxScore/percentage/grade
  - calculateGrade method maps percentage to letter grade
  - Indexes for user/category/status/history queries
- server/models/Question.js
  - Fields: category, subcategory, difficulty, type, question, options, correctAnswer, sampleAnswer, evaluationCriteria, hints, points, tags, usageCount, isAIGenerated
  - Category/difficulty and tags indexing for retrieval

5.5) Controllers
- server/controllers/authController.js
  - Register/login/google auth and current-user lookup
- server/controllers/interviewController.js
  - startInterview:
    - Enforces only one in-progress interview per user
    - Supports multi-subcategory generation
    - Uses AI service; falls back to DB when needed
    - Deduplicates questions and retries fill to exact requested count
    - Can return 503 when exact count cannot be met
  - submitAnswer:
    - Validates in-progress state
    - Evaluates answer via AI service
    - Saves per-question score/feedback and updates progress
  - completeInterview:
    - Finalizes status, percentage, grade
    - Generates overall interview feedback
    - Updates user aggregate stats
  - abandonInterview:
    - Marks abandoned and records duration
  - getInterview/getInterviews:
    - Single interview details and paginated history
- server/controllers/questionController.js
  - Category metadata
  - Filtered paginated question retrieval
  - Single-question lookup
- server/controllers/userController.js
  - Profile read/update, password change, account delete
- server/controllers/analyticsController.js
  - Returns overview, category stats, recent performance, weekly progress, difficulty distribution
- server/controllers/leaderboardController.js
  - Top users ranking and current user rank

5.6) Middleware
- server/middleware/authMiddleware.js
  - protect: JWT verification and req.user assignment
  - admin: role guard
- server/middleware/errorMiddleware.js
  - notFound and normalized API error responses
  - Handles Mongoose cast/validation, duplicate keys, JWT errors

5.7) Validators
- server/validators/validators.js
  - Joi schemas for: register, login, updateProfile, startInterview, submitAnswer, changePassword
  - validate(schema) middleware returns 400 on invalid payloads

5.8) AI Service
- server/services/aiService.js
  - Integrates Google Generative AI
  - Model rotation/fallback among configured models
  - JSON parsing + normalization for AI responses
  - generateQuestions:
    - Category/difficulty-aware prompting
    - Technical interview constraints for verbal response suitability
    - Optional user context insertion
    - Deterministic fallback question pools if AI unavailable
  - evaluateAnswer:
    - AI scoring and qualitative feedback
    - Score normalization and concept coverage analysis
    - Fallback keyword-based scoring without AI
  - generateInterviewFeedback:
    - Summarizes strengths, improvement areas, recommendations
    - Deterministic fallback summary if needed

5.9) Seed Data

7) Authentication and Session Behavior
- JWT-based auth for protected routes
- Token set in client storage and attached on API requests
- 401 responses trigger logout/redirect flow on client
- Google OAuth path verifies token server-side before user creation/login

8) Interview and Scoring Logic
- Question count controlled by user (validated range)
- AI-generated and DB-backed fallback questions
- Per-question scoring in 0-10 range
- Interview percentage and grade computed from aggregate score
- Final completion updates persistent user stats (average/best/category scores)

9) Media and Accessibility Behavior
- Camera/microphone permission requested pre-session
- Live camera shown during interview where applicable
- Speech-to-text input available where browser supports Web Speech API
- Graceful fallback to text-only when media/speech unavailable

10) Security and Reliability
- Helmet security headers
- Express Mongo sanitize for payload hardening
- CORS origin restrictions via CLIENT_URL
- API-wide rate limiting
- Joi validation and centralized error shaping
- AI fallback behavior when key is missing or model fails
